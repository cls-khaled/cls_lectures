
============= Call Stack ========
يتم تخزين كافة الفانشكن في الجافا سكريبت داخل ال
Call Stack
تمهيدا لاستدعاء تلك الفانكشن بنفس الترتيب 

بمعني اخر يعتبر قائمة من المهام البرمجية التي تنفذ بنفس الترتيب 
وعند الانتهاء من تنفيذ كل فانكشن يتم حذفها من ال 
Call Stack


01- makeSandwitch();
02- getBread();
03-addTomatoes();
04-addVegtables();

ملحوظة : لو الفانشكن لا تحتوي علي 
return
تقوم بعمل 
return
افتراضي بعد الخروج من الفانشكن

==================== Web API =========
عبارة عن مجموعة من الفانكشن والايفينت التي تقوم المتصفحات 
بتوفيرها للغة الجافا سكريبت للتتمكن لغة الجافا سكريبت من تنفيذ مهامها 
علي سبيل المثال في مثالنا فانكشن
SetTimeOut

=========== Call Back Queue  ==========
مكان في الذاكره يقوم بتخزين المهام البرمجية من فانكنش مختلفة تم توفيرها
من جانب ال 
Web API
بعد اكتمال عملية تنفيذ تلك الفانكشن في ال 
Web API 
يتم نقلها الي ال 
Call Back Queue

علي سبيل المثال فانكشن ال 
SetTimeOut
بعد انتهاء الزمن المحدد لها وليكن 5 ثواني يتم تقلها  من ال
Web API
الي ال 
Call Back Queue

================== Event Loop =======
هو من يقوم بنقل الفانشكن من ال 
call back Queue
الي ال 

Call  Stack

وهو عبارة عن كود برمجي يقوم صناعة فحص دائم لل 
call back Queue
اذا وجد به فانكشن جديدة يقوم نقلها الي ال 
call Stack
لو وجد ال 
Call Stack
فاضي 

01- makeSandwitch();
02- getBread();
03-addTomatoes();
04-addVegtables();
05- chicken is ready 
06- 5-Add Chicken To Sandwich

كل ما سبق هو ما يجعل لغة الجافا سكريبت لغة من نوع
Single threaded          &         Non-Blocking  I/O
برغم ان الجافا سكريبت تعمل في البروسيور على 
one thread
الا انه عندما يكون فاضي يتم ملائة بشكل مباشر  بالفانكشن الموجوده في ال 
Call Back Queue
من خلال ال 
Event Loop
